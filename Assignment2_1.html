The Tower Newspaper Website Evaluation

Step 1

Input and Output Selection

Input Output
Click on heading "Arts and Entertainment" Redirects to new webpage in site.
In "Archives" sidebar: select "March 2020" Redirects to new webpage showing all archives articles from March, 2020.
Click on heading "Subscribe" Opens new page in different tab to subscribe to the paper.
Search "politics" in the search bar. Pulls up all article headlines with the word "politics" in it.

Step 2

Heuristic Evaluation

  1. Visibility of System Status
    1. Findings: When loading a page it takes a second to load the buffer wheel, creating a weird effect where when you see the buffer symbol you know that page is a second from loading, it gives the impression that the wait time is for loading the buffering wheel, rather than the actual web page.
    2. Suggestion:Changing the buffering symbol to one that may load faster than the page.
  2. Match Between System and Real World
    1. Findings:The format and language are a match for the real world, aka CUA.
    2. Suggestions:None.
  3. User Control and Freedom
    1. Findings:Website Supports "Undo" and "Redo" in the search bar. It also supports backing out of a selected option before the page loads. Unfortunately, there are no exit links on the page, relying only on the browsers return button.
    2. Suggestions:Add ways of redirecting to the webpage.
  4. Consistency and Standards
    1. Findings:The search bar does not conflate two words. Very simple exact word search. Relevancy is hit-or-miss.
    2. Suggestions:Refining the search tool to reach industry standards.
  5. Error Prevention
    1. Findings:No error prevention, no warnings.
    2. Suggestions:Add warnings.
  6. Recognition Rather than Recall
    1. Findings:The dropdown menu and search bar are on each page of the website, making it easy to navigate from one place to another without having to backtrack to the home page. Everything is labelled well and would not need recall to maneuver through.
    2. Suggestions:None.
  7. Flexibility and Efficiency of Use
    1. Findings: Allows for keyboard shortcuts, but the overall design is not flexible. Search is straightforward but lacks more sophisticated searching.
    2. Suggestions:The basic and simple nature of the website makes it difficult for me to find any suggestions. Again, the search function needs to be updated.
  8. Aesthetic and Minimalist Design
    1. Findings:The design hits you in the face and the sidebar archive on every page is unnecessary and looks tacked on rather than a cohesive part of the page. You can scroll down forever because of it rather than keeping the format compressed along with the rest of the page. The archive also reoccurs at the bottom of the page, making the page even longer and adding more redundancy.
    2. Suggestions: Move the archive side bar and foot text to having its own separate page that is linked in the menu bar. Or alternatively, keep the sidebar, get rid of the reoccuring archive in the footer, and have the page load more articles to fill in the blank space.
  9. Help Users Recognize, Diagnose, and Recover from Errors
    1. Findings:When searching, there is no spell check to catch spelling errors.
    2. Suggestions: Add a "Did you mean...?" note and link to alternate spelling search.
  10. Help and Documentation
    1. Findings:There is no help documentation or separate page for a user to use.
    2. Suggestions:Again, because of the nature of the site being very simple, it is hard to recommend suggestions. To follow industry standards, however, there should be a help page that can be easily found and linked to on each page of the site.
